# 影響範囲調査スキル

このスキルは、リファクタリング対象シンボルの影響範囲を調査する定型手順を提供します。
メイン会話、researcher サブエージェント、api-explorer サブエージェントなど、どこからでも呼び出せます。

## 入力

- `symbol`: 調査対象のシンボル名 (関数名・クラス名・型名)
- `scope`: 調査範囲 (default: プロジェクトルート)

## 手順

1. `Grep symbol --type-add ...` で全参照を取得
2. 各参照ファイルを `Read` で読み、使用パターンを次の 4 分類に振り分ける
   - 定義側 (1 箇所)
   - 呼び出し側 (関数呼び出し)
   - 型注釈 / シグネチャ参照
   - import / re-export
3. 影響レイヤーを推定する (API / DB / UI / Test / Config)
4. レイヤー × 分類のマトリクスを返す

## 出力形式

```
影響レイヤー: API (12 箇所), Test (8 箇所), UI (3 箇所)
使用パターン:
  - 呼び出し側: 15 箇所
  - 型注釈: 5 箇所
  - import: 3 箇所
推奨修正順序: API → Test → UI
```

## なぜスキルにするか

このスキルは Subagent ネスト禁止の代替パターンとして機能します。
researcher Subagent が「深い API 調査をしたい」と思ったとき、別の Subagent を起動する代わりに、このスキルを呼び出して同じコンテキスト内で深掘りできます。

スキルはメイン会話・Subagent のどちらからでも実行可能で、再利用性が高い手順を集約する場所です。
