## 残した行と理由

### カテゴリ E: プロジェクト固有のコマンド (推測不可能)

- `pnpm vitest run`: テストランナの選択は推測不可能
- `pnpm drizzle-kit push`: マイグレーションコマンドは推測不可能

### カテゴリ F: プロジェクト固有の罠 (非自明な振る舞い)

- `next/image` 必須: コードからは推測しづらい運用判断
- `verifySession()` 経由: 古いコードと混在する場合に Claude が誤判断する
- `drizzle-kit push` の破壊性警告: 一度事故れば永遠の傷

### カテゴリ G: hook 化候補

`fixed/CLAUDE.md` には残していないが、以下は将来的に PreToolUse hook に
昇格させる候補:

- main 直接 push のブロック
- 本番 DB への `drizzle-kit push` 検出ブロック

これらは「常に」「絶対に」型のハード制約なので、CLAUDE.md の
助言 (advisory) よりも hook の強制 (deterministic) のほうが適切。
