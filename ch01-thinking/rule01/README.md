# Rule 01: コンテキストを「有限の予算」として扱う

## このサンプルで何を見せるか

CLAUDE.md は Claude Code セッションの開始時に **毎ターンの先頭にロード** される。つまり、CLAUDE.md に書いた行はすべて、その後の全ターンで「アテンション予算」を消費する税金となる。

「念のため書いておこう」を積み重ねた `bad/CLAUDE.md` (約 480 行) と、ハイシグナルなものだけ残した `good/CLAUDE.md` (約 130 行) を比較することで、本書 Rule 01「コンテキストを『有限の予算』として扱う」を体感する。

## 検証方法

両方の CLAUDE.md を別プロジェクトに配置して Claude Code を起動し、同じ簡単な質問 (例: 「このプロジェクトのテストはどう実行する?」) を投げる:

1. `bad/CLAUDE.md` 版: 起動時 token 消費が大きく、関係ない情報 (社内 wiki 全文・過去議事録) も同時にコンテキストに乗る
2. `good/CLAUDE.md` 版: 起動時 token 消費が小さく、回答も「test コマンド」に直結する

`/cost` (または `/usage`) コマンドで token 消費の差を確認できる。
